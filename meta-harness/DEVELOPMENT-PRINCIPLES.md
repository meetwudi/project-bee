# Development Principles

Meta Harness development is spec-first.

## Spec-First Development

When a material behavior, workflow, requirement, rule, interface, or engineering practice changes, update the applicable harness spec before changing implementation code.

Implementation code is a materialization of the harness spec. Code changes should be traceable back to the harness content they implement.

If no applicable spec exists, create the appropriate harness doc or ask a human where the spec should live before making the material code change.
